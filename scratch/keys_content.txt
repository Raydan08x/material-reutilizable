55. Footer reutilizable como pie de pagina profesional

Que hace?
Es un componente estructural que se coloca al final de toda pagina web. Proporciona informacion de marca, enlaces de navegacion rapidos, datos de contacto, redes sociales y una barra de copyright. Es fundamental para cualquier sitio profesional porque cierra la jerarquia visual, mejora la navegacion y da credibilidad al proyecto.

Objetivo del componente
Crear un pie de pagina que funcione como cierre coherente de la interfaz, que sea totalmente responsive y que se pueda adaptar a cualquier proyecto simplemente cambiando textos, enlaces y colores. En esta pagina se resolvio con un estilo glass sutil que combina con el tema del proyecto.

Estructura general del componente
La interfaz se divide en tres zonas principales. La zona superior usa una grid de tres columnas: marca con descripcion, enlaces de navegacion y datos de contacto. La zona inferior es una barra separada por borde que muestra copyright y una etiqueta informativa. Todo esta contenido dentro de un footer con fondo semitransparente y borde superior.

HTML minimo que debes copiar

El contenedor principal usa la etiqueta semantica footer con clase site-footer y id siteFooter. El id es opcional si no necesitas manipularlo con JavaScript, pero es util para aplicar estilos especificos o animaciones de entrada.

La grid superior usa site-footer__grid con display: grid. En escritorio usa tres columnas: 2fr para la marca y 1fr para cada una de las otras dos columnas. Esto da mas espacio a la descripcion y distribuye equitativamente los enlaces.

La columna de marca site-footer__brand contiene el mismo logo o marca que la navbar. El span brand__mark usa el mismo estilo circular que el header para mantener coherencia visual. El parrafo muted usa la clase existente del proyecto para descripciones secundarias.

La columna de enlaces site-footer__links usa un elemento nav semantico con aria-label para accesibilidad. Dentro hay enlaces a que apuntan a secciones internas con hash o rutas externas. Los enlaces internos se conectan automaticamente con el router de secciones si tu proyecto usa el patron data-section.

La columna de contacto site-footer__contact usa enlaces a para mailto, GitHub y LinkedIn. El atributo target=_blank y rel=noreferrer son obligatorios para enlaces externos.

La barra inferior site-footer__bottom usa dos span: uno para copyright con el simbolo y ano, y otro para una etiqueta informativa como el numero de funcionalidades o version del proyecto.
